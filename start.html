开始使用
三步启动

你写规格，Argus 写论文。

用 Argus 做研究，你不写论文本身——你写一段 MISSION（任务规格），说清你要什么、什么算赢；启动 daemon，它就自治跑完从定题到 PDF 的整条流水线。下面三步。

第 1 步 · 这才是"你写"的部分

写一段 MISSION

核心是一句贡献句 X / Y / Z / W + 范围 + 门禁 + 完成标准。说清楚边界，剩下交给它。

# Mission — <你的题目>

## 贡献句
- X: 我做了什么（一个 benchmark / 一个机制 / 一篇分析）
- Y: 现有工作的缺口（为什么值得做）
- Z: 我的证据（哪些模型 × 哪些数据，跑出什么）
- W: 所以呢（对领域的含义）

## 范围     模型 / 数据 / 预算 / GPU 限制（train-free？只用 GPU 0-2？）
## 硬门禁   页数 ≥ 8、引用 ≥ 35、每个数命中已发表 baseline ±10pp…
## 完成标准 paper/main.pdf 出、门全绿、证据 bundle 齐
## 不做     多卡、微调、合成数据冒充真实…
写 MISSION 的纪律：用数字、立观点、说清不做什么。范围越清楚，agent 越不会跑偏。
第 2 步 · 一条命令

启动 daemon

把 MISSION 喂给守护进程，它就开始自治运转：research → plan → benchmark → run → analysis → draft → review → submission。

# 模型路由的 key（coproxy）必须在环境里，否则 planner 起不来
export COPROXY_KEY=coproxy OPENAI_API_KEY=coproxy

argus-skill --daemon --continuous \
  --project-root <你的项目目录> \
  --objective "$(cat MISSION.md)"
需要 ~/.argus-skill/special_prompts/ 里至少有一个 *.md 操作规则文件（GPU 纪律、诚实约束等），否则新版会拒绝启动。
第 3 步 · 你只管盯

监督，别动手

agent 自己写代码、跑实验、自我评审。你做的是看进度、按需给方向：

# 看实时进度（每个 agent 在干什么、每步产物）
打开 dash.argusbot.cn

# 给一条临时指令（进 agent 的 inbox）
argus-skill --notify "优先把图 4 收尾，别再迭代 layout"

# 紧急停：让它收尾、不开新探索
argus-skill --notify "STOP"
胜负定义永远外置：你能改它的工作方式（special_prompts、notify），但碰不到"什么算赢"——冻结的 scorer 在 agent 够不到的地方。这是它能诚实的原因。
在哪跑

跑在你自己的机器上

Argus 是个本地守护进程，跑在你的 GPU 机器上。想把产出（报告 / 看板 / 论文）发布到自己的域名，照 部署 runbook 一条命令搞定（Cloudflare 隧道 + systemd，无需公网 IP）。
