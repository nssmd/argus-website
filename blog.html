A research lab that actually runs itself

LifeSupervisor
  ├─ Manager        choose the track, move the project
  ├─ Planner        decide what to do next
  └─ SkillLoop
       └─ Engineer ↔ Reviewer

The outer LifeSupervisor keeps the backlog, budget, state, and next wave of work moving. Inside each mission, the SkillLoop has an Engineer write code and run experiments, then a Reviewer decide whether to continue, finish, or change direction. The Manager owns intake and stages; the Planner keeps scheduling forward. Four independent roles, one shared project memory.

When existing skills are not enough, a Scientist distills a new project skill from the task; the Reviewer then updates skills and the research wiki from real usage. The system is now roughly 170k lines of Python with 2,600+ tests — not a prompt wrapper, but a runtime built for long-horizon research.

From a blank topic to 38 research PDFs

Argus turns paper research into one continuous pipeline: research → plan → benchmark → run → analysis → draft → review → submission. It finds the direction, builds the benchmark, runs experiments, analyzes results, makes figures, writes the paper, takes Reviewer rejection, and keeps going.

Give Argus an objective and compute — then get out of the way. It reads, hypothesizes, builds, measures, critiques, and starts the next round. Research no longer waits for tomorrow morning.
