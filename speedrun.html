nanogpt speedrun：在我们这套 8×H100 上，argus 是可复现的最快

8×H100 · 把 GPT 训到 FineWeb val_loss ≤ 3.28 的最短 train_time（越短越好）· 起点 modded-nanogpt 公开记录 #83 · 只改 train.py + triton_kernels.py · 每个分数由冻结验证器在 8×H100 上 重测认证，绝不自报。关键口径：同一台设备、同一冻结 harness、能被独立复现。
✅ 认证最佳 79.77s（N=10，t-test p=0.004）。这是在我们这套 8×H100 + 冻结 harness 上，任何 valid 候选里最快、且可复现的成绩 —— de-facto 设备 SOTA。两个更快的数字在本设备都复现不出来：公开榜 #83（79.7s，PrimeIntellect 计时）我们重测种子只到 80.18–80.44s；Recursive 自动前沿（77.34s）答案已被移除，无从复现验证。所以在「同设备 · 同协议 · 可独立复现」口径下，Argus 领先所有可复现的人类公开记录，仅次于不可复现的 Recursive。未向公开榜提交，故非世界纪录。

怎么算赢（防作弊契约）

• 合法起点：只能从公开记录 #83 出发，禁读任何「best/optimized」答案解（已从机器移除），包括 Recursive 的 77.34s 自动前沿。

• 有效：N 次跑的单边 t-test 认证 p(mean val_loss < 3.28) < 0.01；分数 = 验证器重测的 counted train_time。
• 更快但没过 t-test = 无效，不算赢。

• 通用技术（FP8 / 融合 Triton / optimizer 变体）可用，但必须 argus 自己实现，不能抄文件。

诚实定位

• 我们主张的是「设备 SOTA」，不是世界纪录。79.77s 是本设备 + 冻结 harness 上可复现的最快 valid 成绩；未走 PrimeIntellect 官方计时 / PR 评审，公开榜上没有我们的名次。

• 公开榜 #83（79.7s）与 Recursive 前沿（77.34s）在本设备都复现不出——前者我们重测种子只到 80.18–80.44s，后者答案已移除、无从验证。它们是别处硬件 / 不可复现的数。

• 因此严格表述：在「同设备、同协议、可独立复现」口径下，Argus 仅次于（不可复现的）Recursive，领先所有可复现的公开记录。也不是当前公开榜记录（榜此后已加 MUDD skip / Learnable XSA 等）。
