What · 我们做什么

不是「自动写论文的工具」，
是一台为难题而生的发明引擎

Argus 的本事不在「会写论文」，而在把一个高密度智能任务一刻不停地执行数天：不空转、不打滑、不卡死，而且答案是它自己探索 / 发明出来的，不是查来的。论文只是任务链最长、最能压测整套机制的那一种，不是产品身份。

特性 01

需要创新

答案不在任何地方现成摆着，必须在一个巨大的解空间里搜索、试错、发明——不是一条已知流程能套出来的活。

反例：查个事实、跑个已知脚本——那是工具的活，不需要发明。
特性 02

长程

一次做不完。要数小时到数天、成百上千轮「假设 → 实验 → 度量 → 改」的密集迭代才有结果。

反例：一问一答、改一个 bug——普通 agent 就够，长跑机器是杀鸡用牛刀。
使能条件

可验证

有一个不能造假的诚实指标说「更好了 / 成了」——一个数、一个 benchmark、一个评审裁决。这是它能自主跑、产出能被信任的前提。

为什么关键：没有诚实裁判，就得有人盯着，自主性归零；有了它，买家能自己复验。

一个随手能用的判定器

  1. 有没有一个不能造假的硬指标说「更好了 / 成了」？
  2. 是不是一次做不完、要连续迭代数小时到数天？
  3. 答案要探索 / 发明出来，而不是查得到、套得出？
三个都 yes → 这是 Argus 的菜。 指标靠「品味」→ 不是；一次能做完 → 不是；照流程抄就行 → 不是。
Why · 为什么偏偏面向这类任务

因为这块交集，又小、又值钱、又没人服务好

「需要创新 + 长程 + 可验证」的任务，恰好卡在三方都失灵的地方——这正是 Argus 的护栏天生擅长、而别的方案天生跑不住的形状。

🧑‍🔬

人做：又贵又慢

要连续数天密集迭代、上千次试错。顶尖研究员的时间太贵，也熬不住这种重复密度。

🤖

通用 agent：跑不住

长跑必死于三件事——空转、失忆打滑、卡死活锁；还会 reward hacking 自欺，把假结果当真。

✅

但它自带裁判

有诚实指标 → 可以完全自主跑、且输出可被独立验证。这是别的难题没有、而这类任务独有的礼物。

于是它落进一个真空：人服务不起、通用 agent 服务不了、但又因为可验证而能被一台机器可信地服务。Argus 就是为填这个真空造的。

应用场景 · 同一套原则，多种难题

一个核，五种真实任务

它们不是「都跟研究有关」，而是在「创新 + 长程 + 可验证」上长得一模一样——所以能用同一个引擎驱动，对着各自的真实强基线。

GPU / 基础设施

CUDA Kernel 优化

任务：把一个算子逼近硬件「光速」(SOL) 极限——没有现成最优解，要在 tiling / 调度 / 指令组合里发明。
Argus 怎么起作用：自主迭代「改写 → 真机 profile → 度量」，主动随机化评测输入堵掉「硬编码输入统计量」式作弊，把有效优化沉淀成 skill。
给谁用：AI 基础设施 / 训练团队、算力厂、芯片公司——榨干每一张卡。
基准：vs torch.compile ≈ 0.547 SOL
前沿模型实验室

训练提速 / 训练配方发明

任务：固定硬件上把达标墙钟时间压到最短，或固定预算下把 loss 压到最低——本质是不断发明新的训练 trick。
Argus 怎么起作用：自主跑全部条件、复现强基线、强制用真实算子（不许假 shim），沉淀「什么真的有效」的干净轨迹。
给谁用：做 pretraining / post-training 的实验室、模型训练团队。
基准：8×H100 达标 77.3s · nanochat val_bpb
量化 / 资管 · 最强变现

量化因子挖掘 & 策略研发

任务：从市场数据里发明出有效、样本外稳健的 alpha 因子并组合——因子构造是创造，不是套模板。
Argus 怎么起作用：自主挖掘 + 回测，内建泄漏探针 / 样本外 / 可复现审计链堵住前视与过拟合自欺，产出 reviewer 认证的因子报告；你的私有数据天然抗同质化。
给谁用：量化基金、券商自营、资管——一座 7×24 不知疲倦的因子工厂。
引擎：已接真实 A 股 qlib 回测
研究机构 / 企业 R&D

自动科研 → 过审论文

任务：从 idea 到能扛过同行评审的论文 / 技术报告——最长、最考验整套机制的任务链。
Argus 怎么起作用：8 阶段全流程，每阶段一个对抗式 reviewer 把关，强制 idea 否决日志反平庸、每条 claim 溯源到证据、堵 pilot / 合成数据冒充。
给谁用：研究机构、企业研究院、要系统化产出的团队。
量表：按 EMNLP / AAAI 真实评审
而且它是可扩展的。任何「有诚实硬指标 + 要连续探索数天 + 答案得发明出来」的问题，接成一个新垂域≈在注册表加一行——编译器 pass 调优、算法 / 数据结构优化、有打分器的材料 / 分子设计、系统参数与协议调优、有胜率的博弈策略……都是同一台引擎的下一个垂域。
How · 怎么做到的

跑通一次谁都会，
难的是连续数天不退化

长跑的自治 agent 会死在三件事上。我们把它们从结构上堵死——有默认值、有 200+ 测试兜底，不是 prompt 哄劝。

🟢

不空转

长实验配独立监督进程自看护，agent 不守着它、转去拉别的活；待办空了自动补；一群并行 agent 从任务板领活。

🟡

不打滑

会话被反复有损压缩会「失忆—重读」原地空想（正常轮 ~200 万 token，失忆环涨到 500–700 万）→ 用硬上限小工作记忆 + 会话定期换班根治。

🔴

不卡死

语义进度看门狗（看「项目有没有真前进」而非「进程挂没挂」），停滞自动升级交回 Planner 重拆，卡死 worker 硬超时杀掉改派。

三个会思考的模型协作，谁也不替谁做判断：

L4 · 规划
Plannerbacklog 空了排新任务；还差认证时把「完成」改派成认证任务。
L1 · 工程
Engineer单轮执行：搜文献、写代码、跑实验、写作；和 reviewer 共享工作目录。
L2 · 审稿
Reviewer对照 stage 清单裁决 done / continue / blocked。它是「做完没」的唯一事实来源。

而它怎么啃下需要专业 know-how 的 OOD 难题？靠自进化，不靠「换个更大模型」：

缺能力
当场长一个没现成 skill 就在线蒸馏，联网看「从业者怎么解」+ 本机实测，写成实操手册
每个任务后
自动更新成功→强化 · 没命中→蒸馏 · 有教训→修订 · 连败→退役
失败也
沉淀被否决的方向留下可复用 skill / benchmark / 审计，喂给下一次
结果
越跑越会第 N 次碰同类难题，手里已攥着前 N−1 次攒下的 know-how
Why us · 为什么是我们

我们不是在演示这个系统，
是在用它改它自己

一个能 7×24 自治长跑、还能反过来淬炼并写回自身能力的系统，证明的不是「我们会写 agent」，而是「我们已经把它跑到了能自我复利的临界点」——这种东西攒出来，需要在真机上长跑很多个昼夜。

01

它在自主修改自己的源码

每次任务成功结束，系统把 agent 淬炼出的新能力分类、写回源码并自动 git commit。跑任务 → 长出新能力 → 写回出厂 → 下次启动再带回来，是个闭合的自进化环。

证据：manager 收拾钩子 + git log 里一串自治产出的 commit
02

它扛得住生产规模

systemd 7×24 部署（crash / 重启自动恢复、停机排空到任务边界不硬杀）、GPU keep-alive 防云主机回收空卡、并行 teammate 舰队从共享任务板领活。断 SSH、回收 GPU、进程卡死都摧不垮它。

证据：deploy/ 的 systemd unit + 实测在跑的多进程舰队
03

对标顶级实验室的公开数字

同一类基准（kernel SOL、训练墙钟、nanochat），我们用一个开源、可部署、领域无关的引擎去对标——而不是一个买不起也打不开的黑箱。

证据：5 个 verticals/ 真实存在，各跑各的真实基线
04

诚信是结构，不是口号

只用真实公开 benchmark、主动堵 reward hacking、基线同硬件同脚手架复现、全程可复现审计链、绝不报喜报早。这让我们的产出可被独立验证地诚实。

证据：随机化评测输入 / 泄漏探针 / 全规模证据门
Why now · 为什么是现在

瓶颈刚刚换了位置

基座模型在 2025–2026 跨过了「能自主连续做多步」的门槛。于是真正的瓶颈，从「模型能不能做对一步」转移到了「系统能不能数天不退化地一直做下去」——正是我们解的那件事。

🧠
模型够强了，但单靠模型不够

一步的智能已经到位；连续数千步不退化、不作弊，是系统工程问题，不是「再大一号模型」能解的。

💰
这个品类刚被重金验证

估值数十亿美金的实验室正用真金白银证明「自治跑可验证长程任务」是真品类、真值钱——市场已被教育。

🧪
可验证轨迹成了最稀缺资产

带 ground-truth 奖励的研究轨迹，是训练 / RL 下一代模型最缺的燃料，人人在抢。Argus 把它当副产品产出。

⚡
算力太贵，ROI 从未如此高

当一张卡一小时都是钱，「榨干每张卡 / 不让自治 agent 空转」的回报，比任何时候都直接。
