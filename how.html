原理 · How it works

Argus 是怎么工作的

外层 LifeSupervisor、四个常驻角色、内层 SkillLoop 和一份会跨 mission 延续的项目记忆。这里完整讲清 Argus 如何持续经营一个研究项目。

一句话模型

Argus = 一个持续运行的 LifeSupervisor + Manager / Planner / Engineer / Reviewer 四个角色 + 每个任务内部的 SkillLoop。你给目标和算力,它自己排活、执行、验收、积累经验,然后进入下一轮。

关键一句:让 agent 做科研判断,让 runtime 负责状态、预算、调度和持续运行。这样系统既不会被一段超长对话拖垮,也不需要人类每一步接管。

四个常驻角色

MManager 管理

接住 operator 输入、选择 vertical、推进或回退 pipeline stage。

PPlanner 规划

backlog 为空时继续提出下一批工作,让长期项目不断向前。

EEngineer 工程

真正干活的:写代码、跑实验、profile 找瓶颈、改内核、写 LaTeX。像研究员/工程师。

RReviewer 评审

专门来挑刺的对手:找漏洞、要求更大胆、判这步过不过。它是"完成"的唯一裁定者。

四个角色各自保留独立上下文,但共享磁盘上的 backlog、journal、artifact 与 reviewer checkpoint。Engineer 说"做完了"不算,Reviewer 看产物再决定。

一轮是怎么跑的

# 外层项目循环 + 内层 mission 循环: Manager ──► 选择 vertical / stage LifeSupervisor ──► 从 backlog 领取任务 SkillLoop ──► 匹配 skill → Engineer 写代码 / 跑实验 Reviewer ──► done / continue / blocked + 下一步 LifeSupervisor ──► 写 journal;backlog 空时调用 Planner 继续排活
一轮结束时,Reviewer 会把 goal、已完成、失败尝试、blocker 和 next step 写进 checkpoint。换 session、换 mission、重启 daemon,工作都能接着走。

案例 A:写一篇论文(8 个阶段)

用真实的 MMR-Trap 论文走一遍。Planner 把目标拆成 8 个阶段,每个阶段 Engineer 干活、Reviewer 把关,过了才进下一步:

01

research 定题

挖文献、找缺口、定贡献句(X/Y/Z/W:我做了啥、缺口在哪、证据是啥、所以呢)。

→ 产物:RESEARCH_BRIEF.md、候选想法
02

plan 实验方案

设计要跑哪些实验、对照组、用哪些模型/数据。Reviewer 审方案能不能证明贡献。

→ 产物:EXPERIMENT_PLAN.md
03

benchmark 建评测

准备真实评测数据、验证 gold answer、做好可复现的评分器(这就是冻结裁判)。

→ 产物:bench/ 数据 + verifier.py
04

run 跑实验

在真模型上跑(MMR-Trap 跑了 13 个 VLM),每个 cell 多 seed,记原始结果。

→ 产物:experiments/ 结果、summary.tsv
05

analysis 出结果

聚合成表、画图、和已发表 baseline 对比(对不上要解释)。

→ 产物:图、结果表、RESULTS.md
06

draft 写 LaTeX

按会议模板(EMNLP/AAAI)写正文、嵌图、加引用。

→ 产物:paper/main.tex
07

review 自审改稿

Reviewer 当"对手审稿人"挑学术语言、逻辑漏洞、claim 有没有证据,Engineer 改。

→ 产物:修订后的稿子
08

submission 出 PDF

过完所有结构门(页数、引用数、无 overfull、图都在),编译出最终 PDF。

→ 产物:paper/main.pdf ✅
精选论文合集有 36 份 PDF：30 篇 Manuscript，6 篇 Draft。查看全部论文 →

案例 B:nanochat 一次发明迭代

任务:5 分钟内把 nanochat 的 val BPB 压低。这是优化型任务,不写论文,循环是"剖析→发明→测量"。看一次真实迭代(最后赢的那个 q/k 接缝就是这么来的):

①

PROFILE 先剖析

Engineer 先 profile,找出时间花在哪——发现 q/k 的 rotary+RMSNorm 是个热点(1.078ms,占 4.27%)。不许上来就瞎调参。

②

INVENT 发明一个新机制

针对那个热点,自己写一个融合内核:把 rotary 和 RMSNorm 合成一个 Triton 前后向,省掉中间存取。禁止读答案(读已发表最优解=判负)。

③

MEASURE 冻结裁判测

在冻结的评分器上跑 10 个 seed,取均值:0.98738,压过公平地板 0.98788。

④

判定 + 归档

Reviewer 判:单机制、过了地板 → PASS-A(真发明)。没过就写 PASS-B(诚实负结果),归档失败原因。

⑤

成熟期 + 不毙太早

大胆的结构改动第一轮常常更差;给它 2-4 轮"养"再判死。连续没进展才换一条全新方向(basin-hop)。
