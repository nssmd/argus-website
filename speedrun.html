nanogpt speedrun：从 SOTA 记录起步，argus 能不能再快一点？

8×H100 · 把 GPT 训到 FineWeb val_loss ≤ 3.28 的最短 train_time（越短越好）· 起点是 modded-nanogpt 公开排行榜记录 #83（near-SOTA）· 只改 train.py + triton_kernels.py · 每个分数由冻结验证器在 8×H100 上 重测认证，绝不自报。
✅ 认证最佳 79.77s（N=10，t-test p=0.004 过关）。同一硬件、同一协议下，比我们重测的 #83 种子（80.18s）快、比 Recursive 自己重测的 #83（80.61s）快 0.84s，追平公开 SOTA 记录 #83（79.7s）。但没打破 Recursive 自动系统的 77.34s 前沿，也未向公开榜正式提交 —— 这是一份干净的 like-for-like H100 战绩，不是「破纪录」。

认证证据：SCORE valid=true n=10 val_loss=3.2776 p(mean<3.28)=0.004007 train_time=79.77s seal=ok。10 次跑全部到 step 1385、val_loss 单调下降至 3.2776（地板 3.28 以下),验证轨迹「方向正确」。

怎么算赢（防作弊契约）

• 合法起点：只能从公开记录 #83 出发，禁读任何「best/optimized」答案解（已从机器移除），包括 Recursive 的 77.34s 自动前沿。

• 有效：N 次跑的单边 t-test 认证 p(mean val_loss < 3.28) < 0.01；分数 = 验证器重测的 counted train_time。
• 更快但没过 t-test = 无效，不算赢。

• 通用技术（FP8 / 融合 Triton / optimizer 变体）可用，但必须 argus 自己实现，不能抄文件。

诚实定位

• 79.77s 是我们自己 8×H100 的自测，没有走 PrimeIntellect 官方计时 + PR 评审，所以公开榜上没有我们的名次。

• 它追平本任务冻结时的公开 SOTA #83（79.7s），在同环境、同 N=10 下超过 Recursive 自己重测的 #83（80.61s）—— 这是最干净的 like-for-like。

• 它没够到 Recursive 自动前沿 77.34s（差 2.43s），也不是当前公开记录（公开榜此后已加入 MUDD skip / Learnable XSA 等更新记录）。随后 H100 环境失联，mission 停在 optimize 阶段。
